# 企业智能办公助手开发手册

这份手册是写给刚开始学编程的人看的。

如果你今年 80 岁，刚接触代码，也可以照着一步一步做。不要急。这个项目不是考试题，它更像一台家用电器。我们只要学会“怎么通电、怎么开机、怎么使用”，就已经很厉害了。

## 1. 这个项目到底是什么

这个项目叫 **企业智能办公助手**。

你可以把它想成公司里的一个聪明前台：

- 员工可以问公司制度
- 员工可以查自己的薪酬
- 员工可以查自己的年假、合同等信息
- 员工还可以让它帮忙整理出差预订信息

你不需要先懂 AI 才能用它。就像你不需要先懂汽车发动机，才会坐车。

## 2. 你会看到哪些东西

这个项目有三部分，像一个小店铺：

### 前端

前端就是“门面”和“按钮”。

你打开浏览器看到的登录页、聊天页、知识库页面，都是前端。

### 后端

后端就是“后台工作人员”。

你在页面上点一下“发送”，后端就开始处理问题，再把结果送回来。

### 数据库和向量库

你可以把它们想成不同的柜子：

- PostgreSQL：放工资、员工信息这些整整齐齐的数据
- Redis：放临时数据
- Milvus：放文档知识，方便以后做知识检索

## 3. 这个项目已经能做什么

现在这个项目已经不是空壳子了。它已经可以：

1. 让你登录系统
2. 展示智能工作台
3. 记录聊天历史
4. 上传和删除知识库文档
5. 查询样例薪酬和个人信息
6. 做商旅信息整理
7. 通过流式输出一段一段返回答案

有些功能会用真实模型，有些功能在没有外部服务时会自动改用本地回退逻辑。这样做的好处是：你先能跑起来，再慢慢接更真实的外部系统。

## 4. 先准备什么

你只需要准备这几样：

1. 电脑
2. 终端
3. Python
4. Node.js
5. Docker

如果你不熟悉这些名字，也没关系。你把它们理解成项目运行要用到的工具箱就行。

## 5. 第一步：复制配置文件

打开终端，进入项目目录后，输入：

```bash
cp backend/.env.example backend/.env
cp frontend/.env.example frontend/.env
```

这两句话的意思很简单：

- 第一份是后端配置，放在 `backend/` 里面
- 第二份是前端配置，放在 `frontend/` 里面

你可以把它理解成“先把空白表格复印一份，再往里面填内容”。

## 6. 第二步：启动配套服务

输入：

```bash
docker compose up -d
```

这句话的意思是：

“把项目需要的数据库和后台服务都启动起来，而且让它们在后台安静运行。”

如果电脑比较慢，第一次启动可能会等一会儿，因为 Docker 需要先下载镜像。

## 7. 第三步：启动后端

输入：

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
PYTHONPATH=backend uvicorn app.main:app --reload --app-dir backend
```

你不用一下子记住每一条命令。

你只要知道，它们在做四件事：

1. 建一个干净的小环境
2. 进入这个环境
3. 安装后端需要的依赖
4. 启动后端程序

启动成功后，浏览器打开：

`http://localhost:8000/api/health`

如果看到 `ok`，说明后端已经正常工作。

## 8. 第四步：启动前端

另外打开一个终端窗口，输入：

```bash
cd frontend
npm install
npm run dev
```

然后打开浏览器：

`http://localhost:5173`

这时候你应该就能看到红色主题的登录页面。

## 10. 登录后先做什么

你可以按这个顺序试：

### 先试知识检索

输入：

`帮我总结公司的差旅报销标准，并列出关键限制条件。`

### 再试个人信息

输入：

`帮我查看我的合同到期日、剩余年假和部门信息。`

### 再试薪酬

输入：

`请查询我本月的薪酬总包。`

### 最后试商旅代办

输入：

`下周二帮我预订上海到深圳的商务舱，2位乘客。`

## 11. 为什么回答会一段一段出现

因为这个项目用了 **SSE 流式输出**。

你可以把它想成：

不是等整封信写完再一起寄给你，
而是一写好一点，就先送一点给你看。

这样用户会感觉系统更快、更自然。

## 12. 为什么有时候不用 OpenAI Key 也能运行

这是项目专门做的“回退方案”。

意思是：

- 如果你已经配置了真实模型，就走真实模型
- 如果你还没配置，也不会整个项目都瘫掉

比如：

- 知识检索可以回退到本地词法检索
- 结构化数据可以回退到本地样例数据库
- 商旅接口可以回退到本地订单确认

这样对于初学者非常友好，因为你不用第一天就把所有外部服务都接完。

## 13. 如果你用的是 MiniMax，不是 OpenAI，怎么办

也可以。

这个项目现在已经支持 **OpenAI 兼容接口**，所以你可以把 MiniMax 当成“长得像 OpenAI 的接口”来接。

打开 `backend/.env` 文件，把这一段改成：

```env
OPENAI_API_KEY=你的 MiniMax Key
OPENAI_BASE_URL=https://api.minimaxi.com/v1
OPENAI_MODEL=MiniMax-M2.7
OPENAI_EMBEDDING_MODEL=
```

这里最后一行故意留空，不是写错了。

因为很多人一开始只有文本模型，没有单独配置 Embedding 服务。留空以后，系统会自动换成本地检索回退方案，这样项目先能跑。

## 14. 如果要接真实商旅系统，怎么做

打开 `backend/.env` 文件，把下面这些项填上：

```env
TRAVEL_API_BASE_URL=https://travel.example.com
TRAVEL_API_PATH=/orders
TRAVEL_API_AUTH_TOKEN=your-token
TRAVEL_API_TIMEOUT_SECONDS=10
TRAVEL_API_FALLBACK_ENABLED=true
```

意思是：

- 外部商旅系统地址是什么
- 下单接口路径是什么
- 要不要带认证令牌
- 最多等多久
- 如果外部接口坏了，要不要自动回退到本地下单

如果你现在还没有真实商旅地址，这几项可以先不填。项目不会坏掉，它会自动用本地订单确认逻辑继续工作。

## 15. 如果知识库页面打不开怎么办

先检查你登录的是不是这两种身份之一：

- `wang.hr`
- `chen.kb`

因为知识库管理页不是所有用户都能看见。

## 16. 如果我只记住三句话

请记住这三句就够了：

1. 前端是你看到的页面，后端是背后处理事情的人。
2. Docker 帮我们一次启动很多服务。
3. 这个项目先保证能跑，再一步一步接入更真实的外部能力。

## 17. 你下一次可以做什么

如果今天你已经能把它跑起来，下一次你可以尝试这些小练习：

1. 换一个账号登录，看看权限有什么不同
2. 上传一份新的 TXT 文档到知识库
3. 修改页面上的文字颜色，观察界面变化
4. 打开后端代码，看看 `backend/app/api/routes` 里有哪些接口

做到这里，你已经不是“完全不会编程的人”了。你已经开始会运行、观察、修改一个真实项目了。
